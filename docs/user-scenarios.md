# 마이링크 사용자 시나리오

## 1. 문서 개요

- 문서명: 마이링크 사용자 시나리오
- 프로젝트명: 마이링크 (MyLink)
- 작성 목적: 방문자와 소유자의 주요 사용 흐름을 정의한다.
- 기준 문서: `docs/prd.md`

---

## 2. 방문자 시나리오

### 2.1. 퍼블릭 페이지 접근

- 사용자는 특정 개발자 또는 크리에이터의 링크를 확인하기 위해 공유받은 마이링크 URL에 접속한다.
- 사용자는 특정 사용자의 프로필을 확인하기 위해 `mylink.com/displayName` 형식의 URL을 입력한다.
- 사용자는 계정 없이 링크 목록을 확인하기 위해 로그인하지 않은 상태로 퍼블릭 페이지에 접근한다.
- 사용자는 모바일 환경에서 링크를 확인하기 위해 스마트폰 브라우저로 퍼블릭 페이지에 접속한다.
- 사용자는 데스크탑 환경에서 링크를 확인하기 위해 PC 브라우저로 퍼블릭 페이지에 접속한다.

### 2.2. 프로필 정보 확인

- 사용자는 페이지 소유자가 누구인지 파악하기 위해 프로필 사진을 확인한다.
- 사용자는 페이지 소유자를 식별하기 위해 닉네임을 확인한다.
- 사용자는 페이지 소유자의 활동 분야를 이해하기 위해 한 줄 소개(Bio)를 확인한다.
- 사용자는 신뢰할 수 있는 페이지인지 판단하기 위해 Google 계정 기반 프로필 이미지를 확인한다.

### 2.3. 링크 목록 확인

- 사용자는 소유자의 주요 채널을 파악하기 위해 링크 블록 목록을 확인한다.
- 사용자는 링크의 목적을 빠르게 이해하기 위해 링크 버튼 제목을 확인한다.
- 사용자는 링크의 출처를 구분하기 위해 링크 버튼의 파비콘 아이콘을 확인한다.
- 사용자는 원하는 작업물을 찾기 위해 GitHub, 블로그, 포트폴리오, 프로젝트 데모 링크를 확인한다.
- 사용자는 원하는 콘텐츠 채널을 찾기 위해 YouTube, Instagram, 블로그 링크를 확인한다.

### 2.4. 링크 클릭

- 사용자는 소유자의 GitHub 작업물을 보기 위해 GitHub 링크를 클릭한다.
- 사용자는 소유자의 기술 글을 읽기 위해 기술 블로그 링크를 클릭한다.
- 사용자는 소유자의 프로젝트 결과물을 보기 위해 프로젝트 데모 링크를 클릭한다.
- 사용자는 소유자의 포트폴리오를 확인하기 위해 포트폴리오 링크를 클릭한다.
- 사용자는 소유자의 콘텐츠를 확인하기 위해 소셜 미디어 링크를 클릭한다.
- 사용자는 소유자에게 연락하기 위해 문의 또는 연락 채널 링크를 클릭한다.

### 2.5. 방문자 예외 상황

- 사용자는 존재하지 않는 `displayName` 페이지에 접근했음을 알기 위해 404 또는 안내 페이지를 확인한다.
- 사용자는 잘못된 외부 링크에 접근했음을 알기 위해 브라우저 또는 외부 사이트의 오류 화면을 확인한다.
- 사용자는 파비콘이 표시되지 않는 링크의 목적을 이해하기 위해 버튼 제목을 확인한다.

## 3. 소유자 시나리오

### 3.1. 서비스 시작

- 사용자는 자신의 링크 페이지를 만들기 위해 메인 랜딩 페이지에 접속한다.
- 사용자는 로그인하지 않은 상태에서 링크 관리가 로그인 이후 제공됨을 알기 위해 안내 문구를 확인한다.
- 사용자는 어느 화면에서든 로그인할 수 있도록 헤더의 "Google 로그인" 버튼을 확인한다.
- 사용자는 서비스를 시작하기 위해 "Google로 시작하기" 버튼을 클릭한다.
- 사용자는 자신의 Google 계정으로 서비스를 사용하기 위해 Google 소셜 로그인을 진행한다.
- 사용자는 대시보드에 접근하기 위해 Firebase Authentication 인증을 완료한다.

### 3.2. 최초 프로필 생성

- 사용자는 초기 프로필을 빠르게 만들기 위해 Google 로그인 이메일의 `@` 앞부분을 `displayName` 초기값으로 사용한다.
- 사용자는 표시 이름의 초기값을 빠르게 설정하기 위해 Google 로그인 정보의 이름을 `userName` 값으로 사용한다.
- 사용자는 프로필 이미지를 별도로 업로드하지 않기 위해 Google 계정의 PhotoURL을 프로필 이미지로 사용한다.
- 사용자는 자신의 퍼블릭 페이지를 만들기 위해 초기 `displayName` 기반 URL을 생성한다.
- 사용자는 최초 프로필의 소개 입력 위치를 확인하기 위해 기본 문구 `한 줄 소개를 입력해주세요.`를 확인한다.

### 3.3. 프로필 확인

- 사용자는 현재 공개되는 정보를 확인하기 위해 대시보드의 프로필 영역을 확인한다.
- 사용자는 계정 메뉴를 확인하기 위해 헤더의 프로필 아바타 버튼을 클릭한다.
- 사용자는 올바른 계정으로 로그인했는지 확인하기 위해 프로필 메뉴의 이름과 이메일을 확인한다.
- 사용자는 자신의 퍼블릭 URL을 확인하기 위해 `displayName` 값을 확인한다.
- 사용자는 방문자에게 표시될 소개를 확인하기 위해 한 줄 소개(Bio)를 확인한다.
- 사용자는 프로필 이미지가 올바른지 확인하기 위해 Google 계정 프로필 사진을 확인한다.
- 사용자는 로그인한 계정이 맞는지 확인하기 위해 사용자 문서에서 불러온 이메일과 표시 이름을 확인한다.

### 3.4. 프로필 수정

- 사용자는 수정 가능한 프로필 정보를 발견하기 위해 표시 이름, 주소 아이디 또는 소개글 텍스트에 마우스를 올려 연필 아이콘을 확인한다.
- 사용자는 프로필 정보를 간단히 수정하기 위해 표시 이름, 주소 아이디 또는 소개글 텍스트를 클릭한다.
- 사용자는 퍼블릭 URL에 사용할 주소 아이디를 관리하기 위해 `displayName` 값을 확인한다.
- 사용자는 방문자에게 보이는 이름을 변경하기 위해 `userName` 텍스트를 클릭하고 새 표시 이름을 저장한다.
- 사용자는 변경한 표시 이름을 다음 방문에도 유지하기 위해 `users/{uid}.userName`에 저장한다.
- 사용자는 퍼블릭 URL의 주소 아이디를 바꾸기 위해 `displayName` 텍스트를 클릭한다.
- 사용자는 사용할 수 있는 URL 형식을 지키기 위해 영문 소문자, 숫자, `.`, `_`, `-`로 주소 아이디를 입력한다.
- 사용자는 URL 아이디의 중복을 방지하기 위해 `중복 확인` 버튼을 클릭한다.
- 사용자는 사용 가능한 주소임을 확인하기 위해 `사용 가능한 주소 아이디입니다.` 안내를 확인한다.
- 사용자는 확인된 주소 아이디를 적용하기 위해 저장 아이콘 버튼을 클릭한다.
- 사용자는 동시에 발생하는 중복 저장을 막기 위해 저장 시 Firestore 트랜잭션으로 주소 예약이 다시 확인되도록 한다.
- 사용자는 프로필 변경 결과를 빠르게 확인하기 위해 Firestore 저장 완료 전 변경된 표시 이름 또는 주소 아이디를 화면에서 확인한다.
- 사용자는 자신을 더 잘 소개하기 위해 한 줄 소개(Bio) 텍스트를 클릭한다.
- 사용자는 소개글을 즉시 수정하기 위해 인라인 편집 상태에서 내용을 입력한다.
- 사용자는 입력한 소개글을 별도 저장 버튼 없이 반영하기 위해 입력을 멈추거나 포커스를 벗어난다.
- 사용자는 소개글 저장 상태를 확인하기 위해 `저장 중` 또는 `저장됨` 안내를 확인한다.

### 3.5. 프로필 수정 예외 상황

- 사용자는 비어 있는 `userName`을 저장할 수 없음을 알기 위해 표시 이름 필수값 안내 메시지를 확인한다.
- 사용자는 프로필 저장이 실패했음을 알기 위해 변경 전 값으로 복구된 화면과 오류 안내를 확인한다.
- 사용자는 이미 사용 중인 `displayName`을 사용할 수 없음을 알기 위해 중복 안내 메시지를 확인한다.
- 사용자는 중복 확인하지 않은 새 `displayName`을 저장할 수 없음을 알기 위해 중복 확인 요청 메시지를 확인한다.
- 사용자는 비어 있는 `displayName`을 저장할 수 없음을 알기 위해 필수값 안내 메시지를 확인한다.
- 사용자는 유효하지 않은 `displayName`을 사용할 수 없음을 알기 위해 형식 안내 메시지를 확인한다.

### 3.6. 링크 추가

- 사용자는 새 외부 채널을 공유하기 위해 링크 목록 상단의 "새로운 링크 추가하기" 버튼을 클릭한다.
- 사용자는 새 링크 정보를 입력하기 위해 표시된 추가 다이얼로그를 확인한다.
- 사용자는 링크 블록을 생성하기 위해 링크 제목을 입력한다.
- 사용자는 외부 페이지로 이동시키기 위해 링크 URL을 입력한다.
- 사용자는 링크 URL을 올바르게 저장하기 위해 `http://` 또는 `https://` 형식의 URL을 입력한다.
- 사용자는 저장이 진행 중임을 확인하기 위해 추가 다이얼로그 제출 버튼의 회전 로딩 아이콘을 확인한다.
- 사용자는 새 링크를 저장하기 위해 자신의 `users/{uid}/links` 경로에 Firestore 문서가 생성되도록 추가 버튼을 클릭한다.
- 사용자는 링크 출처를 시각적으로 보여주기 위해 URL 기반 Google Favicon API 아이콘을 확인한다.
- 사용자는 새 링크가 표시되었음을 확인하기 위해 저장 완료 후 다시 조회된 링크 목록을 확인한다.
- 사용자는 추가 완료를 확인하기 위해 저장 성공 후 닫힌 추가 다이얼로그를 확인한다.

### 3.7. 링크 추가 예외 상황

- 사용자는 링크 제목이 비어 있음을 알기 위해 필수값 안내 메시지를 확인한다.
- 사용자는 링크 URL이 비어 있음을 알기 위해 필수값 안내 메시지를 확인한다.
- 사용자는 링크 URL 형식이 잘못되었음을 알기 위해 URL 형식 안내 메시지를 확인한다.
- 사용자는 파비콘을 불러오지 못했음을 알기 위해 기본 아이콘이 표시되는 것을 확인한다.

### 3.8. 링크 수정

- 사용자는 수정 가능한 링크 정보를 쉽게 찾기 위해 각 링크 카드에 항상 표시된 연필 아이콘을 확인한다.
- 사용자는 기존 링크 정보를 변경하기 위해 링크 아이템의 연필 아이콘을 클릭한다.
- 사용자는 빠르게 내용을 바꾸기 위해 링크 카드 안에서 표시된 제목과 URL 입력 필드를 수정한다.
- 사용자는 변경을 취소하기 위해 인라인 편집 상태의 취소 버튼을 클릭한다.
- 사용자는 변경한 제목과 URL을 저장하기 위해 인라인 편집 상태의 저장 버튼을 클릭한다.
- 사용자는 추가와 동일한 입력 규칙을 지키기 위해 제목 필수값과 URL 형식 검증을 통과한다.
- 사용자는 변경 내용을 보존하기 위해 기존 링크 문서가 Firestore에서 갱신되도록 저장한다.
- 사용자는 수정 결과를 확인하기 위해 저장 완료 후 다시 조회된 링크 목록을 확인한다.
- 사용자는 마지막 변경 시점을 확인하기 위해 수정된 링크 카드의 `수정됨` 시각을 확인한다.
- 사용자는 URL 변경 결과를 확인하기 위해 새 URL 기준의 파비콘 아이콘을 확인한다.
- 사용자는 퍼블릭 페이지의 링크 목적을 명확히 하기 위해 버튼 제목을 수정한다.

### 3.9. 링크 수정 예외 상황

- 사용자는 수정한 링크 제목이 비어 있음을 알기 위해 필수값 안내 메시지를 확인한다.
- 사용자는 수정한 링크 URL이 비어 있음을 알기 위해 필수값 안내 메시지를 확인한다.
- 사용자는 수정한 링크 URL 형식이 잘못되었음을 알기 위해 URL 형식 안내 메시지를 확인한다.

### 3.10. 링크 삭제

- 사용자는 더 이상 공유하지 않을 링크를 제거하기 위해 카드에 항상 표시된 삭제 버튼을 클릭한다.
- 사용자는 삭제 대상을 확인하기 위해 `정말 삭제하시겠습니까?` 모달에 표시된 링크 이름을 확인한다.
- 사용자는 삭제가 되돌릴 수 없는 동작임을 알기 위해 빨간색 `이 작업은 되돌릴 수 없습니다` 경고를 확인한다.
- 사용자는 링크를 유지하기 위해 삭제 확인 모달의 `취소` 버튼을 클릭한다.
- 사용자는 불필요한 링크를 정리하기 위해 삭제 확인 모달의 `삭제하기` 버튼을 클릭한다.
- 사용자는 삭제를 완료하기 위해 해당 링크 문서가 자신의 `users/{uid}/links`에서 제거되도록 한다.
- 사용자는 삭제 결과를 확인하기 위해 삭제 완료 후 다시 조회된 링크 목록을 확인한다.
- 사용자는 삭제한 링크가 더 이상 보이지 않게 하기 위해 퍼블릭 페이지에서 링크 목록을 확인한다.

### 3.11. 링크 삭제 예외 상황

- 사용자는 삭제 요청이 처리 중임을 확인하기 위해 `삭제하기` 버튼의 회전 로딩 아이콘을 확인한다.
- 사용자는 Firestore 삭제가 실패했음을 알기 위해 오류 메시지를 확인한다.

### 3.12. 퍼블릭 페이지 확인

- 사용자는 자신의 공개 페이지로 이동하기 위해 헤더의 `내 페이지로 바로가기` 버튼 또는 프로필 메뉴의 `내 페이지로 이동`을 클릭한다.
- 사용자는 방문자에게 보이는 결과를 확인하기 위해 프로필 메뉴의 `내 페이지 미리보기`를 클릭한다.
- 사용자는 현재 프로필과 링크 구성을 확인하기 위해 미리보기 다이얼로그의 링크 목록을 확인한다.
- 사용자는 자신의 페이지가 올바르게 공개되는지 확인하기 위해 `mylink.com/displayName` URL에 접속한다.
- 사용자는 프로필 수정 결과를 확인하기 위해 퍼블릭 페이지의 닉네임과 소개글을 확인한다.
- 사용자는 링크 추가 결과를 확인하기 위해 퍼블릭 페이지의 링크 블록 목록을 확인한다.
- 사용자는 링크 수정 결과를 확인하기 위해 퍼블릭 페이지의 버튼 제목과 이동 URL을 확인한다.
- 사용자는 링크 삭제 결과를 확인하기 위해 삭제한 링크가 퍼블릭 페이지에서 사라졌는지 확인한다.

### 3.13. 링크 클릭 조회수 확인

- 사용자는 어떤 링크가 많이 눌렸는지 확인하기 위해 개별 링크의 클릭 조회수를 확인한다.
- 사용자는 링크별 관심도를 파악하기 위해 각 링크 블록의 클릭 횟수를 확인한다.
- 사용자는 전체 링크 성과를 보기 위해 `/stats` 통계 페이지에서 총 클릭 수를 확인한다.
- 사용자는 링크별 클릭 분포를 비교하기 위해 `/stats` 통계 페이지의 차트를 확인한다.
- 사용자는 로그인하지 않은 상태에서 통계 페이지에 접근할 수 없음을 알기 위해 메인 페이지로 리다이렉트된다.

### 3.14. 로그아웃

- 사용자는 자신의 계정 사용을 종료하기 위해 프로필 메뉴의 `로그아웃`을 클릭한다.
- 사용자는 다른 사람이 대시보드에 접근하지 못하게 하기 위해 로그아웃한다.

### 3.15. 내 페이지 링크 복사

- 사용자는 자신의 페이지를 공유하기 위해 프로필 메뉴의 `내 페이지 링크 복사`를 클릭한다.
- 사용자는 복사된 개인 페이지에 방문자가 바로 접근할 수 있도록 현재 서비스 도메인의 `/{displayName}` 주소를 공유한다.
- 사용자는 복사 완료 여부를 확인하기 위해 Sonner 성공 또는 실패 토스트를 확인한다.
