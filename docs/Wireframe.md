# 마이링크 와이어프레임

## 1. 문서 개요

- 문서명: 마이링크 와이어프레임
- 프로젝트명: 마이링크 (MyLink)
- 작성 목적: PRD와 사용자 시나리오를 바탕으로 주요 UI 구조를 정의한다.
- 기준 문서: `docs/prd.md`, `docs/user-scenarios.md`, `docs/link-management.md`
- 표현 방식: Mermaid 다이어그램

## 3. 랜딩 페이지 구조

```mermaid
flowchart TB
  subgraph Page["랜딩 페이지"]
    Header["상단 영역: MyLink 로고"]
    Hero["히어로 영역: 개발자와 크리에이터를 위한 링크 관리 서비스 소개"]
    CTA["주요 버튼: Google로 시작하기"]
    Note["보조 문구: 흩어진 링크를 하나의 페이지로 공유"]
  end

  Header --> Hero
  Hero --> CTA
  CTA --> Note
```

### 구성 요소

- 로고: `MyLink`
- 서비스 설명: 개발자와 크리에이터가 작업물, 포트폴리오, 소셜 미디어 링크를 한 곳에 모아 공유하는 서비스
- CTA: `Google로 시작하기`
- 인증 방식: Firebase Authentication 기반 Google 소셜 로그인

## 5. 대시보드 화면 구조

```mermaid
flowchart TB
  subgraph Dashboard["대시보드 화면"]
    TopBar["상단 바: MyLink / 퍼블릭 페이지 보기 / 로그아웃"]

    subgraph Profile["프로필 영역"]
      Photo["프로필 사진: Google photoURL, 수정 불가"]
      Name["닉네임: displayName + 연필 아이콘"]
      Bio["한 줄 소개: bio + 연필 아이콘"]
      Url["퍼블릭 URL: mylink.com/@displayName"]
    end

    subgraph Links["링크 목록 영역"]
      Add["상단 강조 버튼: + 새로운 링크 추가하기"]
      AddDialog["추가 다이얼로그: 제목 / URL / 취소 / 추가 및 로딩 아이콘"]
      Item1["링크 아이템: 파비콘 / 제목 / URL / 항상 표시되는 수정 및 삭제 버튼"]
      Item2["링크 아이템: 파비콘 / 제목 / URL / 항상 표시되는 수정 및 삭제 버튼"]
      Empty["빈 상태: 아직 등록된 링크가 없습니다"]
    end
  end

  TopBar --> Profile
  Profile --> Links
  Add --> AddDialog
  Add --> Item1
  Item1 --> Item2
```

### 구성 요소

- 프로필 사진은 Google 계정 이미지를 사용하며 수정하지 않는다.
- 닉네임과 한 줄 소개 옆에는 연필 아이콘을 표시한다.
- 닉네임과 한 줄 소개는 연필 아이콘 또는 텍스트 클릭으로 인라인 편집한다.
- 링크 목록 최상단에는 아이콘 하나와 문구만 담은 강조된 `새로운 링크 추가하기` 버튼을 표시하고 선택 시 추가 다이얼로그를 연다.
- 링크 목록은 링크 블록 단위로 표시한다.
- 링크 블록은 파비콘, 제목, URL, 새 탭 열기 버튼, 항상 표시되는 연필 아이콘 및 삭제 버튼으로 구성한다.
- 모바일 뷰 미리보기는 제공하지 않는다.

## 6. 프로필 인라인 편집 구조

```mermaid
stateDiagram-v2
  [*] --> ViewMode
  ViewMode: 텍스트 표시 상태
  EditMode: input 또는 contenteditable 상태
  Saving: 저장 중
  Saved: 저장 완료
  Error: 오류 표시

  ViewMode --> EditMode: 텍스트 또는 연필 아이콘 클릭
  EditMode --> Saving: 엔터 키 또는 포커스 아웃
  Saving --> Saved: 저장 성공
  Saving --> Error: 중복 또는 유효성 오류
  Saved --> ViewMode: 변경값 표시
  Error --> EditMode: 수정 재시도
```

### 닉네임 수정 규칙

- `displayName`은 URL slug로 사용한다.
- `displayName`은 비어 있을 수 없다.
- `displayName` 저장 시 유일성 검사가 필요하다.
- `displayName` 변경 시 퍼블릭 URL도 함께 변경된다.

## 8. 링크 아이템 와이어프레임

```mermaid
flowchart LR
  Favicon["파비콘"]
  Content["제목 / URL 인라인 편집 영역"]
  Edit["항상 표시: 연필 아이콘 수정 버튼"]
  Delete["항상 표시: 휴지통 아이콘 삭제 버튼"]
  Open["새 탭 열기 버튼"]

  Favicon --> Content
  Content --> Edit
  Edit --> Delete
  Delete --> Open
```

### 링크 아이템 구성

- 파비콘: `https://www.google.com/s2/favicons?domain={url}` 형식으로 클라이언트에서 동적 생성
- 제목: 방문자에게 보이는 버튼 제목
- URL: 클릭 시 이동할 외부 링크
- 연필 아이콘: 항상 표시하며, 클릭 시 제목과 URL 인라인 수정 모드 진입
- 삭제 버튼: 항상 표시하며, 클릭 시 삭제 확인 모달 표시
- 새 탭 열기 버튼: 현재 링크 URL을 새 탭에서 열기

## 9. 링크 수정 상태 구조

```mermaid
stateDiagram-v2
  [*] --> LinkView
  LinkView: 링크 제목과 URL 표시
  InlineEdit: 제목 및 URL 인라인 입력
  Validate: 추가와 동일한 입력 검증
  Saving: Firestore 수정 저장 중
  UpdateFavicon: 파비콘 다시 렌더링
  LinkSaved: 저장 완료
  LinkError: 오류 표시

  LinkView --> InlineEdit: 항상 표시된 연필 아이콘 클릭
  InlineEdit --> Validate: 저장 버튼 클릭
  InlineEdit --> LinkView: 취소 버튼 클릭
  Validate --> InlineEdit: 입력 검증 실패
  Validate --> Saving: 입력 검증 통과
  Saving --> Reload: Firestore 수정 성공
  Saving --> LinkError: Firestore 수정 실패
  Reload: Firestore 목록 재조회
  Reload --> UpdateFavicon
  UpdateFavicon --> LinkSaved
  LinkSaved --> LinkView
  LinkError --> InlineEdit: 수정 재시도
```

## 10. 퍼블릭 페이지 구조

```mermaid
flowchart TB
  subgraph PublicPage["퍼블릭 페이지: mylink.com/@displayName"]
    ProfilePhoto["프로필 사진"]
    DisplayName["닉네임: displayName"]
    Bio["한 줄 소개"]
    LinkList["링크 블록 목록"]
    LinkA["링크 버튼: 파비콘 + 제목"]
    LinkB["링크 버튼: 파비콘 + 제목"]
    LinkC["링크 버튼: 파비콘 + 제목"]
  end

  ProfilePhoto --> DisplayName
  DisplayName --> Bio
  Bio --> LinkList
  LinkList --> LinkA
  LinkList --> LinkB
  LinkList --> LinkC
```

### 퍼블릭 페이지 규칙

- 방문자는 로그인 없이 접근한다.
- URL은 `mylink.com/@displayName` 형식을 사용한다.
- 프로필 이미지는 Google 계정 이미지를 사용한다.
- 링크 블록은 버튼 형태로 표시한다.
- 각 링크 버튼은 파비콘과 제목을 함께 표시한다.
- 링크 클릭 시 외부 사이트로 이동한다.

## 13. 추후 링크 클릭 조회수 위치

```mermaid
flowchart LR
  LinkItem["링크 아이템"]
  ClickCount["클릭 조회수"]

  LinkItem --> ClickCount
```

### 적용 범위

- 방문자 통계 또는 분석 대시보드는 포함하지 않는다.
- 추후 개별 링크 블록에 클릭 조회수만 표시할 수 있다.
