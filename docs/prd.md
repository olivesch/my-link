# 마이링크 PRD

## 1. 문서 개요

- 문서명: 마이링크 PRD(기능 정의서)
- 프로젝트명: 마이링크
- 문서 목적: 개발자와 크리에이터가 자신의 주요 링크를 하나의 퍼블릭 페이지로 공유할 수 있는 서비스의 기능 범위를 정의한다.
- 작성 범위: 프로젝트 개요, 필수 기능 목록, 기능 상세 설명, 데이터베이스 모델링 방향
- 제외 범위: 사용자 시나리오, 와이어프레임, 상세 UI 설계, 개발 일정
- 참고 계획: 사용자 시나리오와 와이어프레임은 이후 `docs` 폴더에 별도 문서로 작성한다.

## 2. 프로젝트 개요

### 2.1 프로젝트명

마이링크

### 2.2 프로젝트 목적

마이링크는 개발자와 크리에이터가 GitHub, 포트폴리오, 기술 블로그, SNS, 콘텐츠 채널, 문의 링크 등을 하나의 페이지에 모아 공유할 수 있는 링크트리 클론 서비스다.

사용자는 여러 외부 링크를 각각 공유하지 않고, 자신의 정체성과 활동을 보여주는 하나의 대표 URL을 공유할 수 있다.

### 2.3 대상 사용자

- 개발자: GitHub, 포트폴리오, 기술 블로그, 프로젝트 데모 링크를 정리하려는 사용자
- 크리에이터: YouTube, Instagram, 블로그, 뉴스레터 등 여러 채널을 한 곳에 모으려는 사용자
- 취업 준비생: 이력서, 포트폴리오, GitHub, 블로그를 간단히 공유하려는 사용자
- 프리랜서: 작업물, 문의 채널, SNS 링크를 한 페이지로 전달하려는 사용자

### 2.4 핵심 가치

- 여러 링크를 하나의 URL로 공유할 수 있다.
- 개발자와 크리에이터의 활동 채널을 간단하고 명확하게 보여준다.
- 방문자는 로그인 없이 퍼블릭 페이지에서 링크를 확인할 수 있다.
- 사용자는 링크 블록을 직접 추가, 수정, 삭제할 수 있다.

## 3. 현재 전제

- 첫 버전은 웹 기반 서비스로 제작한다.
- MVP는 링크 공유 경험에 집중한다.
- 로그인한 사용자만 자신의 프로필과 링크 블록을 관리할 수 있다.
- 방문자는 로그인 없이 퍼블릭 페이지를 볼 수 있다.
- 링크 아이템의 기본 아이콘은 해당 URL의 파비콘을 사용한다.
- 방문자 통계 및 분석 대시보드는 MVP에 포함하지 않는다.
- 링크 클릭 조회수는 추후 링크별 단순 수치로 추가할 수 있다.
- 모바일 뷰 미리보기 기능은 제공하지 않는다.

## 4. 필수 기능 목록

| 구분 | 기능명 | 설명 | 우선순위 |
| --- | --- | --- | --- |
| 필수 | 사용자 인증 | 사용자가 회원가입, 로그인, 로그아웃을 할 수 있다. | High |
| 필수 | 프로필 관리 | 사용자가 퍼블릭 페이지에 표시될 프로필 정보를 관리한다. | High |
| 필수 | 링크 블록 관리 | 사용자가 링크 블록을 추가, 수정, 삭제하고 버튼 제목과 URL을 설정한다. | High |
| 필수 | 퍼블릭 페이지 | 방문자가 사용자 프로필과 링크 블록 목록을 볼 수 있다. | High |

## 5. MVP 제외 기능

아래 기능은 현재 PRD의 필수 범위에서 제외한다.

- 방문자 통계 및 분석 대시보드
- 모바일 뷰 미리보기
- 테마 커스터마이징
- QR 코드 공유
- 커스텀 URL 고도화
- SNS 아이콘 링크 별도 관리
- 링크 활성/비활성
- 링크 정렬 고도화
- 팀 또는 협업 기능
- 결제 기능

단, 링크 클릭 조회수는 이후 링크 블록 단위의 단순 수치로 추가할 수 있다.

## 6. 기능 상세 설명

### 6.1 사용자 인증

사용자는 계정을 생성하고 로그인한 뒤 자신의 프로필과 링크 블록을 관리할 수 있어야 한다.

#### 요구사항

- 사용자는 이메일과 비밀번호로 회원가입할 수 있다.
- 사용자는 이메일과 비밀번호로 로그인할 수 있다.
- 사용자는 로그아웃할 수 있다.
- 로그인하지 않은 사용자는 관리 화면에 접근할 수 없다.
- 로그인한 사용자는 본인의 데이터만 조회, 수정, 삭제할 수 있다.

#### 예외 상황

- 이미 가입된 이메일로 회원가입을 시도하면 중복 안내를 표시한다.
- 비밀번호가 최소 조건을 만족하지 않으면 가입을 막는다.
- 로그인 정보가 올바르지 않으면 오류 메시지를 표시한다.
- 세션이 만료되면 다시 로그인하도록 안내한다.

### 6.2 프로필 관리

사용자는 퍼블릭 페이지에 표시될 자신의 기본 정보를 관리할 수 있어야 한다.

#### 관리 항목

- `displayName`: 퍼블릭 페이지에 표시될 이름
- `username`: 퍼블릭 페이지 URL에 사용될 고유 이름
- `bio`: 짧은 소개글
- `profileImageUrl`: 프로필 이미지 URL

#### 요구사항

- `displayName`은 필수 입력값이다.
- `username`은 필수 입력값이며 중복될 수 없다.
- `bio`는 선택 입력값이다.
- `profileImageUrl`은 선택 입력값이다.
- 프로필 정보 저장 후 퍼블릭 페이지에 반영되어야 한다.

#### 공개 URL 예시

```text
https://mylink.example.com/u/username
```

### 6.3 링크 블록 관리

사용자는 퍼블릭 페이지에 표시할 링크 블록을 추가, 수정, 삭제할 수 있어야 한다.

#### 링크 블록 입력 항목

- 버튼 제목
- URL

#### 링크 블록 표시 항목

- 파비콘 아이콘
- 버튼 제목

#### 요구사항

- 사용자는 링크 블록을 추가할 수 있다.
- 사용자는 기존 링크 블록의 버튼 제목과 URL을 수정할 수 있다.
- 사용자는 링크 블록을 삭제할 수 있다.
- 버튼 제목과 URL은 필수 입력값이다.
- URL은 `http://` 또는 `https://`로 시작하는 유효한 URL이어야 한다.
- 링크 블록의 기본 아이콘은 URL의 도메인에서 가져온 파비콘을 사용한다.
- 파비콘을 가져오지 못한 경우 기본 링크 아이콘을 표시한다.
- 링크 블록은 생성된 순서 또는 저장된 순서 기준으로 퍼블릭 페이지에 표시한다.

#### 링크 블록 예시

```text
버튼 제목: GitHub
URL: https://github.com/username
기본 아이콘: github.com의 favicon
```

### 6.4 퍼블릭 페이지

방문자는 로그인 없이 사용자의 퍼블릭 페이지를 볼 수 있어야 한다.

#### 표시 항목

- 프로필 이미지
- 표시 이름
- 소개글
- 링크 블록 목록

#### 요구사항

- 방문자는 로그인 없이 퍼블릭 페이지에 접근할 수 있다.
- 퍼블릭 페이지는 `username` 기반 URL로 접근한다.
- 퍼블릭 페이지에는 사용자의 프로필 정보와 링크 블록 목록이 표시된다.
- 각 링크 블록은 파비콘과 버튼 제목을 함께 보여준다.
- 링크 블록을 클릭하면 해당 URL로 이동한다.
- 존재하지 않는 `username`으로 접근하면 404 또는 안내 페이지를 표시한다.
- 모바일, 태블릿, 데스크탑에서 자연스럽게 볼 수 있는 반응형 UI를 제공한다.

## 7. 관리자 화면 구성 방향

관리자 화면은 사용자가 자신의 프로필과 링크 블록을 빠르게 수정할 수 있도록 단순하게 구성한다.

### 주요 영역

- 프로필 정보 편집 영역
- 링크 블록 목록 영역
- 링크 블록 추가 버튼
- 퍼블릭 페이지 보기 버튼

### 설계 원칙

- 가장 중요한 작업은 링크 블록 추가, 수정, 삭제다.
- 링크 블록의 버튼 제목과 URL을 빠르게 수정할 수 있어야 한다.
- 모바일 뷰 미리보기는 제공하지 않는다.
- 복잡한 설정은 MVP 화면에 포함하지 않는다.

## 8. 퍼블릭 페이지 구성 방향

퍼블릭 페이지는 방문자가 사용자의 정체성과 주요 링크를 빠르게 이해할 수 있어야 한다.

### 주요 영역

- 프로필 이미지 영역
- `displayName` 표시 영역
- 소개글 영역
- 링크 블록 목록

### 설계 원칙

- 링크 블록은 클릭 가능한 버튼 형태로 제공한다.
- 각 링크 블록에는 기본적으로 해당 링크의 파비콘을 표시한다.
- 모바일 화면에서도 버튼이 충분히 크고 누르기 쉬워야 한다.
- 방문자의 링크 클릭 흐름을 방해하는 요소를 최소화한다.

## 9. 데이터베이스 모델링

데이터베이스는 사용자 문서 하위에 링크 블록 서브 컬렉션을 두는 구조를 사용한다.

### 9.1 컬렉션 구조

```text
users/{userId}
users/{userId}/links/{linkId}
```

### 9.2 users 문서

사용자의 인증 정보와 퍼블릭 프로필 정보를 저장한다.

| 필드 | 타입 | 설명 | 필수 여부 |
| --- | --- | --- | --- |
| id | string | 사용자 고유 ID | 필수 |
| email | string | 로그인 이메일 | 필수 |
| username | string | 퍼블릭 페이지 URL에 사용되는 고유 이름 | 필수 |
| displayName | string | 퍼블릭 페이지에 표시되는 이름 | 필수 |
| bio | string | 짧은 소개글 | 선택 |
| profileImageUrl | string | 프로필 이미지 URL | 선택 |
| createdAt | timestamp | 생성일 | 필수 |
| updatedAt | timestamp | 수정일 | 필수 |

### 9.3 users/{userId}/links 서브 컬렉션

사용자별 링크 블록 목록을 저장한다.

| 필드 | 타입 | 설명 | 필수 여부 |
| --- | --- | --- | --- |
| id | string | 링크 블록 고유 ID | 필수 |
| title | string | 버튼 제목 | 필수 |
| url | string | 이동할 링크 URL | 필수 |
| faviconUrl | string | 해당 링크의 파비콘 URL | 선택 |
| order | number | 링크 블록 표시 순서 | 필수 |
| createdAt | timestamp | 생성일 | 필수 |
| updatedAt | timestamp | 수정일 | 필수 |

### 9.4 파비콘 처리 기준

- 링크 블록 생성 또는 URL 수정 시 URL의 도메인을 기준으로 파비콘을 설정한다.
- 파비콘 URL을 가져올 수 있으면 `faviconUrl`에 저장한다.
- 파비콘을 가져오지 못하면 `faviconUrl`은 비워둘 수 있다.
- `faviconUrl`이 없는 링크 블록은 UI에서 기본 링크 아이콘을 표시한다.

### 9.5 추후 클릭 조회수 확장

방문자 통계 및 분석 대시보드는 현재 범위에서 제외한다.

다만 추후 링크별 클릭 조회수를 추가할 경우 `users/{userId}/links/{linkId}` 문서에 아래 필드를 추가할 수 있다.

| 필드 | 타입 | 설명 |
| --- | --- | --- |
| clickCount | number | 해당 링크 블록의 누적 클릭 수 |

## 10. 비기능 요구사항

### 10.1 성능

- 퍼블릭 페이지는 빠르게 로드되어야 한다.
- 링크 블록 목록은 모바일 네트워크 환경에서도 지연을 최소화해야 한다.
- 파비콘 로딩 실패가 페이지 전체 렌더링을 막지 않아야 한다.

### 10.2 보안

- 비밀번호는 평문으로 저장하지 않는다.
- 사용자 입력값은 검증한다.
- 다른 사용자의 프로필과 링크 블록을 수정할 수 없도록 권한을 검증한다.
- URL은 `http://` 또는 `https://` 형식만 허용한다.

### 10.3 접근성

- 링크 블록은 키보드로 접근 가능해야 한다.
- 버튼 텍스트는 명확해야 한다.
- 프로필 이미지는 대체 텍스트를 제공한다.
- 파비콘만으로 링크 목적을 전달하지 않고 버튼 제목을 함께 표시한다.

## 11. 오픈 질문

아래 항목은 구현 전에 결정이 필요하다.

- 회원가입 방식을 이메일/비밀번호만 사용할지, 소셜 로그인을 추가할지
- 공개 URL을 `/u/username` 형태로 할지, `/username` 형태로 할지
- 프로필 이미지를 직접 업로드할지, 외부 이미지 URL 입력으로 시작할지
- 링크 블록 순서 변경을 MVP에 포함할지, 생성 순서대로만 표시할지
