## 1. 프로젝트 개요

---

### 1.1. 프로젝트명

**마이링크 (MyLink)**

### 1.2. 목적

개발자와 크리에이터가 자신의 작업물, 포트폴리오, 소셜 미디어 등 흩어진 디지털 자산을 통합하여 관리하고 공유할 수 있는 서비스를 제공한다.

### 1.3. 대상 사용자 (Target Audience)

- **Primary:** 자신의 기술 블로그, Github 작업물, 프로젝트 데모 등을 효과적으로 보여주고 싶은 개발자.
- **Secondary:** 포트폴리오 관리가 필요한 크리에이터.

## 2. 핵심 기능 목록 (Feature List)

---

### 2.1. 필수 기능 (Must-Have / MVP)

1. **사용자 인증 (Auth)**
   - Firebase Authentication 기반 **Google 소셜 로그인**.
   - 비로그인 헤더에는 로그인 버튼을, 로그인 헤더에는 아바타 프로필 버튼을 제공한다.
   - 프로필 메뉴에 사용자 이름/이메일, 내 페이지 이동, 내 페이지 미리보기, 내 페이지 링크 복사, 로그아웃을 제공한다.
   - 로그인 전에는 링크 관리 기능 대신 로그인 후 이용 가능한 기능을 안내한다.

2. **프로필 관리**
   - **DisplayName 기반 URL:** `displayName`을 URL slug로 사용 (예: `mylink.com/displayName`).
   - **DisplayName 초기값:** Google 로그인 이메일의 `@` 앞부분을 가져와 설정 (예: `devkim@gmail.com` -> `devkim`).
   - **UserName 초기값:** Google 계정에 등록된 이름을 가져와 프로필 표시 이름으로 설정하며 이후 사용자가 수정할 수 있다.
   - **텍스트 클릭 편집 UI:** 표시 이름, 주소 아이디, 소개글 텍스트에 마우스를 올리거나 키보드 포커스를 두면 연필 아이콘을 표시하고, 텍스트를 클릭하면 즉시 수정 모드로 전환한다.
   - 수정 가능 항목: 표시 이름(`userName`), URL 주소 아이디(`displayName`), 한 줄 소개(Bio). 사진은 Google 계정 이미지를 사용한다.
   - **DisplayName 중복 확인:** 변경할 주소 아이디는 저장 전 중복 확인을 거치며, 저장 시에도 Firestore 트랜잭션으로 점유 여부를 재확인한다.
   - **소개글 기본값:** 신규 사용자 또는 이전 기본 문구를 가진 사용자의 `bio`를 `한 줄 소개를 입력해주세요.`로 저장하고 표시한다.
   - **낙관적 프로필 저장:** 표시 이름, 주소 아이디, 한 줄 소개는 저장 요청 전에 화면에 변경값을 먼저 반영하고, Firestore 저장이 실패하면 직전 값으로 되돌리고 오류를 표시한다.
   - **소개글 저장:** 한 줄 소개 입력 중 화면에 즉시 반영하고, 입력이 멈추거나 포커스를 벗어나면 `users/{uid}.bio`에 저장한다.

3. **링크 블록 관리**
   - 링크 추가/수정/삭제.
   - **추가 다이얼로그:** 목록 상단의 강조된 `새로운 링크 추가하기` 버튼을 선택하면 제목과 URL 입력 다이얼로그 표시.
   - **상시 표시 관리 버튼:** 각 링크 카드에 연필 아이콘 수정 버튼과 휴지통 아이콘 삭제 버튼을 항상 표시.
   - **인라인 편집:** 연필 아이콘 클릭 시 링크 제목 및 URL을 카드 안에서 바로 수정.
   - **공통 입력 검증:** 링크 추가와 수정에 동일한 제목 필수값 및 URL 형식 검증 적용.
   - **삭제 확인 모달:** 링크 이름과 되돌릴 수 없다는 경고를 확인한 뒤 삭제 실행.
   - **파비콘 자동 연동 (Google API):** 링크 URL 입력 시 Google Favicon API를 사용하여 아이콘 자동 표시.

4. **퍼블릭 페이지 (뷰어)**
   - 방문자가 보는 실제 프로필 페이지를 `/{displayName}` 동적 라우트로 제공한다.
   - `displayName`으로 사용자 문서를 검색하고 `users/{uid}/links`에서 해당 사용자의 링크를 불러온다.
   - 검색된 사용자 문서에 `userName`이 없거나 사용자가 존재하지 않으면 404 페이지를 표시한다.
   - 반응형 디자인 (모바일 최적화).
   - shadcn/ui 기반의 깔끔한 UI.
   - 소유자는 헤더 프로필 메뉴에서 현재 프로필과 링크 구성을 미리 볼 수 있다.

### 2.2. 추후 제공 예정 (Future Plan)

- **링크 클릭 조회수:** 개별 링크의 클릭 횟수 카운팅 기능.

### 2.3. 제외 기능 (Out of Scope)

- **이미지 업로드:** 커스텀 프로필 이미지 업로드 불가 (Google 계정 이미지 사용).
- **테마 시스템:** 다크/라이트 모드 또는 컬러 커스터마이징 없음.
- 링크 활성화/비활성화 토글.
- 링크 드래그 앤 드롭 순서 변경.
- 개발자 특화 카드.
- 별도의 모바일 기기 프레임 미리보기.

## 3. 기능 상세 설명

---

### 3.1. 인증 (Authentication)

- **진입점:** 헤더의 `Google 로그인` 버튼과 로그인 전 화면의 `Google로 시작하기` 버튼.
- **로직:** Firebase Auth.
- **초기값:** Google 로그인 이메일의 `@` 앞부분을 `displayName` 초기값으로 설정하고, Google 계정 이름을 `userName`, Google 계정 사진(PhotoURL)을 프로필 이미지로 설정한다. 저장된 `userName`이 존재하면 Google 값으로 덮어쓰지 않는다.
- **사용자 문서:** 로그인한 사용자별로 `users/{uid}` 문서를 생성 또는 갱신하며, 해당 문서의 `userName`, `displayName`, `photoURL`, `email`, `bio` 정보를 대시보드에 표시한다.
- **비로그인 상태:** 링크 관리 화면 대신 `Development in One Link.` 헤드라인과 개발자 정보를 한 페이지로 요약할 수 있다는 안내 화면을 표시한다.

### 3.2. 대시보드 (Admin) - 인라인 에디팅 중심

- **프로필 영역:**
  - 프로필 사진(Google URL, 수정 불가), 표시 이름(`userName`, 수정 가능), 주소 아이디(`displayName`, 수정 가능), 소개글(수정 가능) 표시.
  - 수정 가능한 프로필 정보는 호버 또는 키보드 포커스 시 연필 아이콘을 표시하고, 텍스트를 클릭하면 `input`으로 전환되는 인라인 편집을 제공한다.
  - **표시 이름 저장:** `userName`은 공백이 아닌 40자 이하의 값으로 저장하며, `users/{uid}.userName`을 갱신한다.
  - 한 줄 소개는 `한 줄 소개를 입력해주세요.` 기본값에서 시작하며, 입력값은 낙관적 저장 상태와 함께 사용자 문서에 반영한다.
  - **주소 형식 검증:** `displayName`은 영문 소문자, 숫자, `.`, `_`, `-`만 사용하며 30자 이하로 입력한다.
  - **중복 검사:** `displayName` 변경 시 중복 확인을 제공하고, 저장 시 `displayNames/{displayName}` 점유 문서를 트랜잭션으로 확인 및 등록하여 유일성을 보장한다.
  - **실패 롤백:** 프로필 저장이 실패하면 먼저 표시된 변경값을 저장 전 값으로 복구하고 편집 오류를 표시한다.

- **헤더 프로필 메뉴:**
  - 로그인 사용자의 아바타를 버튼으로 표시하고 선택 시 이름과 이메일을 보여준다.
  - 헤더의 `내 페이지로 바로가기` 버튼과 메뉴의 `내 페이지로 이동`을 선택하면 `/{displayName}` 퍼블릭 페이지로 이동한다.
  - `내 페이지 미리보기`를 선택하면 `users/{uid}/links`의 최신 링크를 읽어 방문자 화면 형태의 다이얼로그로 표시한다.
  - `내 페이지 링크 복사`를 선택하면 현재 서비스 도메인의 `/{displayName}` 공유 주소를 클립보드에 복사하고 Sonner 토스트로 성공 또는 실패를 안내한다.
  - `로그아웃`을 선택하면 현재 인증 세션을 종료한다.

- **링크 목록 영역:**
  - `새로운 링크 추가하기` 버튼: 목록 상단에 한 개의 추가 아이콘과 문구만 표시하는 단순한 강조 버튼으로 배치하고, 클릭 시 링크 추가 다이얼로그를 연다.
  - 추가 중에는 다이얼로그를 열린 상태로 유지하고, 다이얼로그 내부 추가 버튼에 회전 로딩 아이콘만 표시한다.
  - 추가 저장이 성공하면 다이얼로그를 닫고, 실패하면 다이얼로그에서 오류를 표시한다.
  - 리스트 아이템:
    - 아이콘: URL 입력 시 `https://www.google.com/s2/favicons?domain={url}` 형식을 이용하여 자동 렌더링.
    - 수정 시각: 수정된 링크에는 Firestore의 `updatedAt`을 기반으로 마지막 수정 시각을 표시한다.
    - 연필 아이콘: 항상 표시되며, 클릭 시 제목 및 URL 인라인 수정 모드로 전환한다.
    - 수정 저장: 링크 추가와 동일한 입력 검증을 통과한 뒤 Firestore 문서를 갱신한다.
    - 삭제 버튼: 항상 표시되며, 클릭 시 삭제 확인 모달을 연다.
    - 삭제 확인 모달: 제목 `정말 삭제하시겠습니까?`, 링크 이름, 빨간색 경고 `이 작업은 되돌릴 수 없습니다`, `취소` 및 `삭제하기` 버튼을 표시한다.

### 3.3. 디자인 시스템 (UI/UX)

- **프레임워크:** shadcn/ui.
- **인터랙션:** 표시 이름과 주소 아이디는 텍스트 클릭 기반 인라인 편집과 명시적인 저장/취소를 제공하며, 주소 아이디에는 중복 확인을 추가한다. 프로필 변경은 낙관적으로 화면에 반영하고 실패 시 롤백한다. 링크 편집은 별도의 관리 아이콘과 저장/취소 동작을 유지한다.
- **위험 동작 안내:** 삭제는 확인 모달과 빨간색 경고 문구를 통해 의도하지 않은 삭제를 방지한다.

### 3.4. 링크 데이터 동기화

- 로그인한 사용자의 링크 저장 경로는 `users/{uid}/links`이다.
- 로그인되지 않은 상태에서는 링크 목록 조회 및 관리 기능을 제공하지 않는다.
- 퍼블릭 페이지는 `displayName`에 일치하는 사용자 문서를 조회하고 해당 문서 ID의 `users/{uid}/links` 링크 목록을 읽기 전용으로 표시한다.
- 페이지 로드 시 TanStack Query를 통해 Firestore에서 링크를 한 번 불러오고 `createdAt` 기준 최신순으로 표시한다.
- 링크 추가, 수정, 삭제가 성공하면 링크 목록 쿼리를 무효화하고 Firestore에서 다시 조회하여 화면에 반영한다.
- Firestore 실시간 구독은 사용하지 않는다.

## 4. 데이터베이스 모델링 (NoSQL - Firestore)

---

### 4.1. Users Collection

```json
{
  "uid": "google_uid_123",
  "email": "dev_kim@gmail.com",
  "userName": "Kim Cheolsu",
  "displayName": "dev_kim",
  "photoURL": "https://lh3.googleusercontent.com/...",
  "bio": "한 줄 소개를 입력해주세요.",
  "createdAt": "timestamp",
  "updatedAt": "timestamp"
}
```

### 4.2. DisplayNames Collection (displayNames/{displayName})

```json
{
  "uid": "google_uid_123",
  "displayName": "dev_kim",
  "updatedAt": "timestamp"
}
```

Note: 주소 아이디 중복 확인과 저장 시 이 문서를 조회하며, 저장 트랜잭션에서 예약 문서와 사용자 프로필을 함께 갱신한다.

### 4.3. Links Sub-collection (users/{uid}/links)

```json
{
  "id": "link_uuid",
  "title": "My Blog",
  "url": "https://blog.example.com",
  "createdAt": "timestamp",
  "updatedAt": "timestamp"
}
```

Note: 파비콘은 DB에 저장하지 않고 클라이언트에서 `url`을 기반으로 동적 생성한다.
