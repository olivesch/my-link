# 마이링크 링크 관리 기능 명세

## 1. 문서 개요

- 기능명: 링크 블록 추가, 수정, 삭제
- 대상 사용자: 자신의 개발 결과물과 콘텐츠 링크를 관리하는 소유자
- 관련 문서: `docs/prd.md`, `docs/user-scenarios.md`, `docs/Wireframe.md`
- 구현 기준: Firebase Firestore를 사용하는 현재 프로토타입
- 사용 조건: Firebase Authentication으로 Google 로그인을 완료한 소유자

---

## 2. 기능 목적

소유자는 자신의 기술 블로그, GitHub 저장소, 프로젝트 데모, 포트폴리오 등의 외부 링크를 한 화면에서 빠르게 관리할 수 있어야 한다. 수정과 삭제 버튼은 링크 카드마다 항상 표시하여 관리 기능을 바로 발견할 수 있도록 한다. 또한 소유자는 관리 카드와 미리보기 화면에서 링크 클릭 수를 확인할 수 있어야 한다.

---

## 3. 데이터 저장 위치

- Firestore 경로: `users/{uid}/links`
- 목록 정렬: `createdAt` 기준 최신순
- 조회 방식: TanStack Query로 목록을 조회하며, 추가/수정/삭제 성공 후 쿼리를 무효화하여 다시 조회한다. Firestore 실시간 구독은 사용하지 않는다.
- 파비콘: 데이터베이스에 저장하지 않으며, 저장된 `url`을 바탕으로 화면에서 동적으로 표시한다.
- 클릭 조회수: 퍼블릭 페이지 또는 관리 화면에서 링크가 열리면 해당 문서의 `clickCount`를 1씩 증가시킨다.

### 링크 문서 구조

```json
{
  "title": "기술 블로그",
  "url": "https://blog.example.com",
  "clickCount": 0,
  "createdAt": "timestamp",
  "updatedAt": "timestamp",
  "lastClickedAt": "timestamp"
}
```

- `createdAt`: 링크를 새로 추가할 때 저장한다.
- `updatedAt`: 링크 제목 또는 URL을 수정할 때 저장한다.
- `clickCount`: 방문자가 퍼블릭 페이지에서 링크를 클릭한 누적 횟수이며, 기존 문서에 값이 없으면 화면에서 `0`으로 표시한다.
- `lastClickedAt`: 마지막 클릭 시각이며, 클릭이 발생한 경우에만 저장한다.

---

## 4. 화면 구성

### 4.1. 링크 추가 카드

- 링크 목록의 가장 상단에 강조된 `새로운 링크 추가하기` 버튼을 표시한다.
- 추가 버튼은 하나의 추가 아이콘과 문구만 중앙 정렬한 단순한 형태로 구성한다.
- 버튼을 선택하면 링크 제목과 URL을 입력하는 다이얼로그를 연다.
- 저장 중에는 다이얼로그를 유지하고, 다이얼로그의 추가 버튼 안에 회전 로딩 아이콘만 표시한다.
- 저장에 성공하면 다이얼로그를 닫고, 실패하면 열린 상태에서 오류를 표시한다.

### 4.2. 링크 카드

- 파비콘, 링크 제목, 도메인, 아이콘과 숫자로만 구성된 클릭 조회수, 새 탭 열기 버튼을 표시한다.
- 링크가 수정된 경우 `updatedAt`을 기반으로 `수정됨` 시각을 카드에 표시한다.
- 각 카드의 수정 버튼과 삭제 버튼은 마우스 오버 여부와 관계없이 항상 표시한다.
- 수정 버튼은 연필 아이콘으로 표시한다.
- 삭제 버튼은 휴지통 아이콘으로 표시한다.

---

## 5. 링크 추가

1. 사용자는 링크 목록 상단의 `새로운 링크 추가하기` 버튼을 선택한다.
2. 시스템은 링크 제목과 URL 입력 필드가 있는 다이얼로그를 표시한다.
3. 사용자는 제목과 URL을 입력하고 추가 버튼을 선택한다.
4. 시스템은 입력값 검증을 통과한 경우 로그인한 사용자의 `users/{uid}/links`에 `clickCount: 0`을 포함한 새 문서를 저장한다.
5. 저장 중에는 추가 버튼에 회전 로딩 아이콘을 표시하고 다이얼로그를 유지한다.
6. 새 문서가 생성되면 링크 목록 쿼리를 무효화하고 Firestore에서 다시 조회하여 최신순 목록에 표시한다.
7. 저장이 완료되면 다이얼로그를 닫는다.

---

## 6. 링크 수정

1. 사용자는 수정할 링크 카드에 항상 표시된 연필 아이콘 버튼을 선택한다.
2. 시스템은 해당 카드 안에서 제목과 URL을 입력 필드로 전환한다.
3. 사용자는 입력 내용을 변경한 뒤 저장 버튼을 선택한다.
4. 시스템은 링크 추가와 동일한 입력 검증을 수행한다.
5. 검증에 성공하면 동일한 Firestore 문서를 수정하고 `updatedAt`을 기록한다.
6. 수정 저장이 완료되면 Firestore 목록을 다시 조회하여 변경된 링크 카드를 표시한다.
7. 시스템은 카드에 마지막 수정 시각을 `수정됨` 정보로 표시한다.
8. 사용자는 취소 버튼을 선택하여 저장하지 않고 표시 상태로 돌아갈 수 있다.

---

## 7. 입력 검증

링크 추가와 링크 수정은 동일한 `zod` 스키마와 `react-hook-form` 기반 검증 규칙을 사용한다.

| 항목      | 규칙                                                      | 오류 안내                                                                                                    |
| --------- | --------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------ |
| 링크 제목 | 공백을 제거한 뒤 한 글자 이상이어야 한다.                 | `링크 제목을 입력해주세요.`                                                                                  |
| URL       | 비어 있을 수 없다.                                        | `URL을 입력해주세요.`                                                                                        |
| URL 형식  | 유효한 URL이며 `http://` 또는 `https://` 형식이어야 한다. | `https://로 시작하는 올바른 URL을 입력해주세요.` 또는 `웹 주소는 http 또는 https 형식만 등록할 수 있습니다.` |

검증에 실패하면 Firestore 저장을 실행하지 않고 해당 입력 필드 아래에 오류 메시지를 표시한다.

---

## 8. 링크 삭제

1. 사용자는 삭제할 링크 카드에 항상 표시된 휴지통 아이콘 버튼을 선택한다.
2. 시스템은 삭제 확인 모달을 표시한다.
3. 사용자가 `삭제하기`를 선택하면 Firestore에서 해당 링크 문서를 삭제한다.
4. 삭제가 완료되면 Firestore 목록을 다시 조회하여 삭제한 링크 카드를 화면에서 제거한다.
5. 사용자가 `취소`를 선택하면 문서를 삭제하지 않고 모달을 닫는다.

### 삭제 확인 모달 문구

- 제목: `정말 삭제하시겠습니까?`
- 내용: 삭제 대상 링크 이름 표시
- 경고: `이 작업은 되돌릴 수 없습니다`
- 버튼: `취소`, `삭제하기`

경고 문구는 위험 동작임을 명확히 알 수 있도록 빨간색으로 표시한다.

---

## 9. 링크 클릭 조회수

1. 방문자는 퍼블릭 페이지에서 원하는 링크 버튼을 클릭하거나, 소유자는 관리 화면에서 링크를 새 탭으로 연다.
2. 시스템은 외부 링크를 새 탭으로 열고, 동시에 해당 링크 문서의 `clickCount`를 원자적으로 1 증가시킨다.
3. 시스템은 `lastClickedAt`에 마지막 클릭 시각을 기록한다.
4. 소유자는 대시보드 링크 목록 또는 미리보기 화면에서 각 링크의 누적 클릭 수를 아이콘과 숫자로 확인한다.
5. 방문자 퍼블릭 페이지에는 조회수를 표시하지 않고 클릭 카운트만 누적한다.
6. 클릭 기록이 실패하더라도 외부 링크 이동은 막지 않는다.

---

## 10. 오류 및 처리 중 상태

- 목록 최초 조회 중에는 로딩 상태를 표시한다.
- 로그인 전에는 링크 목록과 관리 버튼을 표시하지 않고 로그인 안내 화면을 제공한다.
- 링크 추가, 수정, 삭제 결과는 작업 완료 후 목록 쿼리를 무효화하고 Firestore에서 다시 조회하여 반영한다.
- 클릭 조회수 증가는 방문자 페이지 이동을 방해하지 않으며, 실패 시 별도 오류 UI를 표시하지 않는다.
- 링크 추가 저장이 실패하면 목록은 변경하지 않고 오류를 표시한다.
- 링크 추가 중에는 추가 다이얼로그의 제출 버튼에 회전 로딩 아이콘만 표시하고 다이얼로그를 닫지 않는다.
- 수정 저장 중에는 저장 버튼을 비활성화하고 진행 상태를 표시한다.
- 삭제 진행 중에는 모달 버튼을 비활성화하고 `삭제하기` 버튼 내부에 회전 로딩 아이콘만 표시한다.
- Firestore 작업이 실패하면 사용자가 재시도할 수 있도록 오류 메시지를 표시한다.

---

## 11. 완료 조건

- 모든 링크 카드에 수정 및 삭제 버튼이 항상 표시된다.
- 링크 수정은 카드 내부 인라인 입력 UI에서 수행된다.
- 수정된 링크 카드에는 Firestore `updatedAt` 기준의 마지막 수정 시각이 표시된다.
- 링크 카드와 소유자 미리보기에는 Firestore `clickCount` 기준의 누적 클릭 수가 아이콘과 숫자로 표시된다.
- 퍼블릭 페이지에서 링크를 클릭하면 해당 링크의 `clickCount`가 증가한다.
- 추가와 수정에 동일한 입력 검증이 적용된다.
- 삭제 전에 지정된 문구의 확인 모달이 표시된다.
- 추가, 수정, 삭제 결과가 로그인 사용자의 `users/{uid}/links` 데이터와 화면 목록에 반영된다.
